# 老婆:我想用AI写小程序。我:???你不是在撸猫做美甲吗?

## 一个周末的对话

我老婆已经两年没碰过代码了,我都快忘了她以前还是个程序员。

那天晚上她突然说"我想写个小程序"的时候,*哥们*当时正在 `dota2` 三连败呢,庆幸老婆帮我找了个理由逃离开黑班子,抬头看她——她抱着她那好几个月没打开过 1 次的 `mac` 缩在沙发对面,眼神那种认真劲儿,我只在上次她吵着想养小狗的时候见到过。

"小程序?"其实我还有点没缓过来,"你不是在做美甲吗?"

她点头:"就是因为在做美甲,所以想写个小程序。"

### 她的背景

她以前是大厂后端,`Java` 写得很硬。后来生了场病,索性放下键盘,去做一直想做的事——学美甲,开工作室。

这个工作室有点特别:主打"撸猫撸狗做美甲"。客人来了,可以一边撸她养的猫和狗,一边做指甲。氛围很松弛,也很治愈。

生意慢慢起来了,她就开始琢磨怎么做点差异化。

### 她的需求

"我想把整个美甲流程放到小程序上,"她说,"客人可以提前看流程、选款式、约时间。还有我们家那几只猫狗,也想给它们做个介绍页面,让客人提前认识一下。"

"你确定?"我问。

"这不是有 `AI` 嘛,我也玩玩看" 她半开玩笑给我说来着。

![image.png](https://cdn.gooo.ai/web-images/613532674cb03a5181965cfcdbd9f3954a6ccbb908eb6553772e82c7b4af3517)

### 我的想法

我愣了一下。最近在 waytoagi 这个 `AI` 社区上搞七捻八玩了不少东西。`Cursor`、`ClaudeCode`、`Trae`、`Gemini`……其实那个时候回过头看看也有好多可以搞搞编程。

突然被我老婆问起来咋整 `ai` 来开发,我才意识到那些摸鱼时候积累的 `ai` 经验,一直都像是我家里电脑里面那个显卡一样,存在但没发挥用处。

![image.png](https://cdn.gooo.ai/web-images/1ab1aa33e75c1944c54c746b7b24c83e229109183d1e11925a5e22b55c100f50)

我自己就是搞 `AI` 产品的,这点用户需求还是能嗅出来—这事儿,肯定不止她一个人需要。

> ***她需要的,可能也是一小部分人需要的。***

> 大家不缺高深的 `AI` 理论,但缺的是那种活人感的学习成长维度:"这东西到底怎么用?能不能帮我把手里的活儿干完?"

> **这个念头一旦冒出来**,就按不回去了——我决定把这事儿写下来:怎么带一个两年没碰代码的人,用 `AI` 从零搞出一个能跑的项目。

> 我本来是记录下这段过程,如果不小心被你看到了,那算你赚到了哈哈哈哈。

不过,这只是故事的一半。

## 另一个一直想填的坑

更早之前——大概是我准备投简历、刷面试的那段时间——我就想搭一个个人博客。

一方面是放作品集,另一方面是我一直想做一个"有意思"的项目:不是那种模板套一套就结束的个人主页,而是一个**看起来就很想点、很想逛、很想收藏**的东西。

![image.png](https://cdn.gooo.ai/web-images/fc45fa27a0d6f76d07364ae03e3b1fbbf35f9964d55cc2d8209b65dd84ec7846)

这个念头断断续续很多年:想做,但总觉得成本太高;想做,又觉得 ~~"等我有时间/等我前端更熟/等我想清楚再说"~~ → **现在就做**。

![image.png](https://cdn.gooo.ai/web-images/698e3ef5e34cf775d8ff92a4faaa8585eb05bf751f8ff9a5ec983dd25eeaa71c)

直到最近,情况变了。

`Vibe coding` 这股风,加上 `Gemini`、`Claude` 这类模型的前端能力明显强了——我忽然冒出一个非常朴素的疑问:

> ***为什么我不现在就做一个好看、好玩、还能持续进化的博客?***

它不一定"对职业最有帮助",但一定是我会喜欢的东西。

## 两件事撞到一起

老婆的小程序,我的博客——两个项目,同一个时间点冒出来。

我想:干脆一起记录吧。

当我认真想把博客搭起来,就不可避免地要写一些东西:

- 为什么这么设计

- 怎么选技术栈/怎么部署

- 哪些工具真的省时间

- 哪些坑会让你莫名其妙卡一天

这些内容如果只写在本地笔记里,很快就会变成"我自己都找不到"的碎片。

我更希望它像一份持续更新的"建站日志/实验记录":写给未来的自己,也写给那些和我一样——想做点有趣项目、但又不想被信息洪流带着跑的人。

于是公众号就顺便开了。

不是为了追热点,不是为了日更打卡,更不是为了制造焦虑。

我只是想把过程写下来:我做了什么,为什么这么做,效果怎样,翻车在哪里。

## 你在这个实验室能看到什么?

> 工具、新闻、玩法教程啥的,我推荐还是找大佬看测评,多去看看公众号,多去看看推特。而我只会写这些东西:

| 类型 | 你能看到什么 |
| --- | --- |
| 🎮 **工具的隐藏玩法** | 大部分人只会基础操作,但其实有更骚的用法 |
| 🧪 **实验日记** | 从 idea 到能跑的东西,完整记录,包括翻车 |
| 🤖 **AI 牛马日常** | 产品经理和 `AI` 协作的真实吐槽、翻车、惊喜 |
| 📦 **收藏不吃灰** | 那些你存了但从没用过的资源,我来帮你用起来 |
| ⚙️ **工作流实验** | 我日常真实在用的 `AI` 流程,哪些顺、哪些坑 |
| 👋 **带人上手** | 像教我老婆用 `AI` 写代码一样,手把手带你入门 |

只要有这个心,什么时候开始都不晚。
